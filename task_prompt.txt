## 第一部分：搜索人事变动（talent 命令已内置中国公司）

```bash
cd /Users/lzw/Documents/LobsterAI/lzw/ai-talent-tracker
python3 scripts/tavily_search.py talent
```

这会执行 5-8 次搜索，覆盖：
- 国际公司综合搜索（3次）
- 中国公司英文搜索（1次）
- 中国公司中文搜索（1次）
- 余量充足时追加头部公司高管搜索

1. 读取 data/events.json
2. 找出 to_company 为空的离职记录
3. 批量搜索去向：
```bash
python3 scripts/tavily_search.py destinations "人名1" "人名2" "人名3"
```

## 第三部分：更新关键洞察页面（如有人事变动）

如果本次更新添加了重要人事变动事件，需要同步更新 insights.html：

1. 分析新事件的性质和影响，提取关键洞察点
2. 在 insights.html 顶部添加新的洞察条目，包括：
   - 日期标记
   - 洞察类型标签（人才流失警报/重磅挖角/安全关切/创业动态/中国AI等）
   - 标题和详细分析
   - 涉及人员标签
   - 数据来源链接
3. 更新页面顶部的概览卡片统计数字
4. 保持时间线格式与现有条目一致

详细格式参考：INSIGHTS_UPDATE_GUIDE.md

## 数据格式

- id: evt-前缀+编号
- person_name, from_company, to_company, role, event_type
- source_url, date_discovered, date_event, summary
- summary 中文和英文均可，优先中文

## 去重与保存

- 人名+公司+事件类型 去重
- 相同 URL 跳过
- 更新 data/events.json，按 date_event 倒序
- 如有变更，执行：
```
git -c user.name="AI-Tracker-Bot" -c user.email="bot@tracker" add data/events.json
git -c user.name="AI-Tracker-Bot" -c user.email="bot@tracker" commit -m "Daily update: N events (intl+cn) via Tavily"
git push origin main
```

- 如果更新了 insights.html，同时提交：
```
git add insights.html
git commit -m "Daily update: N events + insights"
git push origin main
```

## 注意事项
- 脚本自动处理限额（30次/天），超出使用 web-search 备用
- 优先关注高管、知名研究者、创始人
- 中国公司人事变动可能以中文新闻为主，summary 用中文
- 关键洞察页面更新不是每次必须，只在有重要发现时更新
